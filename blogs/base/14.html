# C）抽象类和接口的区别

抽象类通常用于描述一系列关系紧密的类之间，接口只是用来描述一个类应该具有什么行为？

  • 抽象类是is a，接口是has a。
  • 接口可以被多层实现，而抽象类只能单一继承。
  • 抽象类中可以有实现体，接口中只能有函数的声明。

例如：猫类和狗类，它们都是动物类，它们都实现跑的方法。但是猫类和汽车类，它们也都具有跑的方法，因此它们适用于接口。

# B）非模块

我们需要先理解 TypeScript 认为什么是一个模块。

  • 没有export的JS文件被认为是一个脚本，而不是模块。对于TS来说也是如此。
  • 在一个脚本文件中，遍历和类型会被声明在共享的全局作用域。
  • 如果你有一个文件，现在没有任何import或export，但是你希望它被当作模块处理，添加export {}
  • 这会把文件改成一个没有导出任何内容的模块。

# 2. TS命名空间

TypeScript 有它自己的模块格式，名为 namespaces ，它在 ES 模块标准之前出现。

  • 命名空间在TS早期，称之为内部模块，它将一个模块内部再进行作用域划分，防止命名冲突。
  • 虽然TS没有将命名空间废弃，但是ES模块已经拥有了命名空间的大部分特性。因此更推荐使用 ES 模块，这样才能与 JavaScript 的（发展）方向保持一致。

早期在没有ES模块化的情况下，我们使用TS的命名空间解决变量名冲突。
