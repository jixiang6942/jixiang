# 二，浏览器的渲染原理

# 1. 网页的解析过程

当我们在网页地址栏输入地址回车后发生了什么？

输入地址 -> DNS解析 -> 请求服务器 -> 加载index.html -> 加载css或js

总结：先下载index.html，解析过程中遇到link元素再下载css文件，遇到script元素再下载js文件。

# 2. 浏览器渲染流程

# A）浏览器内核

浏览器内核也成为排版引擎、浏览器引擎，页面渲染引擎或样板引擎。

# B）渲染页面

逐行解析html，遇到link元素去下载(并不会阻塞html解析)，html解析最后生成DOM树，将解析完成的style附加到DOM tree上。

  • 经过layout之后，进行painting，然后再进行展示。

# D）解析二：生成CSS规则

下载和解析CSS时，并不会阻塞DOM Tree的解析，但是如果DOM解析完成了，Style Rules没有生产完，这个时候并不会生成Render Tree。

# E）解析三：构建Render Tree

  • 注意一：link元素不会阻塞DOM Tree的构建过程，但是会阻塞Render Tree的构建
  • 注意二：Render Tree 和 DOM Tree并不是一一对应的关系，有些元素display为none。

# F）布局(layout)和绘制(Painting)

  • layout过程：确定Render Tree上所有节点的大小和位置信息。

# 3. 回流和重绘解析

# A）回流

什么是回流？第一次确定节点的大小和位置，称之为布局(layout)。之后对节点的大小、位置修改重新计算重置为回流。

什么情况下引起回流？

  • 改变DOM结构，比如添加新的节点或者移除节点。
  • 改变了布局，比如修改了width、height、padding、font-size等值。
  • 修改窗口尺寸。
  • 调用getComputed方法获取尺寸、位置信息。

# B）重绘

什么是重绘repaint？第一次渲染内容称之为绘制，之后重新渲染称之为重绘。

什么情况下会引起重绘呢？

  • 修改背景颜色、文字颜色、边框颜色、样式等。

# C）避免回流

回流一定引起重绘，所以回流非常消耗性能，所以在开发中要尽量避免回流。

如何避免回流，如下：

  • 修改样式尽量一次修改，比如通过classText或添加class。
  • 尽量避免频繁地操作DOM，可以在一个DocumentFragment或父元素中将要进行的DOM操作完成。
  • 尽量避免通过getComputedStyle获取尺寸、位置等信息。
  • 对某些元素使用position或fixed，并不是说不会引起回流，而是开销相对较小，因为不会对其他元素造成影响。

# C）VO对象(Variabal Object)

  • 要执行代码就要创建一个执行上下文，全局代码会创建一个全局执行上下文(GEC)。
  • 每一个执行上下文(EC)都会关联一个VO(Variable Object，变量对象)。
  • 变量和函数的声明会添加到这个VO对象中。
  • 全局执行上下文的VO就是GO。

创建执行上下文 -> 将变量和函数存放在VO(变量没有赋值) -> 执行代码，给变量赋值。

# 3. 全局代码执行过程

# 4. 函数代码执行过程

总结：

  • 要执行代码块就要创建一个执行上下文
  • 每一个执行上下文都会有一个VO(Variable Object)。
  • 执行代码前，会创建一个AO，并于VO关连。将变量的声明放入AO中，但是没有赋值。
  • 执行代码，对AO中的变量进行赋值。

  • 执行全局代码，作用域链里只有GO，执行函数代码，函数创建的那一刻已经知道自己的作用域链，执行代码时，会把作用连赋值给执行上下文中的scope chain。

# 3. 闭包的概念

  • 闭包是一个结构体，它存储了一个函数和关联体的环境。
  • 一个函数和对其周围状态的引用捆绑在一起，这样的组合就是闭包。
  • 也就是说，闭包让你可以在一个内层函数中访问到其外层函数的作用域。
  • 在js中，每创建一个函数，闭包就会在函数创建的同时被创建出来。

面试回答：

  • 一个普通函数，如果它可以访问外层作用域的自由变量，那么这个函数和周围环境就是一个闭包。
  • 从广义角度来说，JavaScript中的函数都是闭包。
  • 从狭义的角度来说，JavaScript中一个函数，如果访问了外层作用域的变量，那么它是一个闭包。
